# Projbahn

This repo contains:

- A Django + django-ninja HTTP API for projects, features, tasks, and event logs
- A Django-rendered web frontend for project and task management

## Run the backend

```bash
uv run manage.py runserver 8001
```

Open `http://127.0.0.1:8001/` for the web UI and `http://127.0.0.1:8001/api/docs` for the API docs.

## DSPy MLflow tracing

This project can publish DSPy traces to MLflow, following the DSPy observability guide.

Start an MLflow server:

```bash
uv run mlflow server
```

Enable tracing before starting Django:

```bash
export PROJBAHN_DSPY_MLFLOW_TRACKING_URI=http://127.0.0.1:5000
export PROJBAHN_DSPY_MLFLOW_EXPERIMENT_NAME="Projbahn DSPy"
uv run manage.py runserver 8001
```

Then open `http://127.0.0.1:5000/` and inspect the `Projbahn DSPy` experiment traces generated by feature chat requests.

## Mem0 project knowledge

Feature chat can enrich DSPy prompts with project-scoped feature/task memories stored in mem0 while keeping chat thread/message persistence in SQLite.

The mem0 integration is enabled by default in the app runtime and uses:

- ChromaDB for vector storage
- LM Studio for memory extraction and embeddings

Useful settings:

```bash
export PROJBAHN_MEM0_EMBEDDER_MODEL=text-embedding-qwen3-embedding-0.6b
```

Mem0 uses each project's configured LLM name for its extraction model.
